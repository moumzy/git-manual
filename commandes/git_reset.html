git_reset --soft

Descriptif de la commande détaillé

git reset --soft
Cette commande ne touche pas le fichier d'index ( zone de transit où les nouveaux commits sont préparés) ou même l'espace de travail (les fichiers qui sont en train d'être modifiés) mais réinitialise le dernier commit, comme toutes les options de git reset.
Les fichiers modifiés restent des «Changes to be commited» (ou changement qu'il faut commit), comme dirait si bien.

Syntaxe

git reset --soft

Affichage

Par exemple, pour défaire et refaire un commit

git commit ... git reset --soft HEAD^ edit git commit -a -c ORIG_HEAD

git reset --mixed

Cette commande permet d'enlever de la file d'attente une modification qui attend d'être committé. Les modifications restent intact mais ne sont plus marqué comme étant en attente d'être committé.

Syntaxe

git reset --mixed

Affichage

$ git add Nouveau_dossier.html
$ git status
On branch master
Changes to be committed:
new file : Nouveau_dossier.html
 $ git reset --mixed
$ git status
On branch master
Untracked files :
(use git add < file >..." to include in what will be committed)
Nouveau_dossier.html
nothing added to commit but untracked files present (use "git add" to track)
